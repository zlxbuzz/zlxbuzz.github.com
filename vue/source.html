    # 原理

    src 核心目录： compiler #编译相关 core #核心代码 platforms #不同平台的支持 server #服务端渲染 sfc #vue 文件的解析 shared #公共的方法

    # 数据驱动

    数据初始化： init->initState->proxy->$mount

    $mount 关键就是 mountComponent 函数: 先实例化一个渲染 Watcher，在它的回调函数中会调用 updateComponent 方法，在此方法中调用 vm._render 方法先生成虚拟 Node，最终调用 vm._update 更新 DOM

    Vue 的 _render 会生成一个 vnode,也就是 virtual dom. render: 参数 vm.$createElement template:也会生成 render 函数，最终都会去执行这个 render 函数，他的参数是 vm._c

    vnode = render.call(vm._renderProxy, vm.$createElement);

// initRender
vm._c = (a, b, c, d) => createElement(vm, a, b, c, d, false);
vm.$createElement = (a, b, c, d) => createElement(vm, a, b, c, d, true);

    # 组件化

    createComponent:构造子类构造函数=>安装组件钩子函数=>实例化 vnode 之后会执行 vm.__patch__(最终会调用 core/vdom/patch 的 createPatchFunction 也就是 src/core/vdom/patch.js 中返回的函数)

    patch 流程： createComponent->子组件初始化->子组件 render->子组件 patch activeInstance 为当前激活的 vm 实例；vm.$vnode 为组件的占位 vnode;vm._vnode 为组件的渲染 vnode 嵌套组件的插入顺序是先子后父

    合并配置：new Vue 的时候和子组件初始化

    // 合并配置例子 Vue.options->extends->mixins->new Vue 参数 ->子组件

    子组件初始化过程通过 initInternalComponent 方式要比外部初始化 Vue 通过 mergeOptions 的过程要快，合并完的结果保留在 vm.$options

    // 组件注册

    全局注册:src/core/global-api/assets.js 定义了 Vue.component 局部注册

    异步组件：普通函数异步组件，Promise 异步组件,高级异步组件

    # 响应式

    响应式对象：Object.defineProperty

    Observer: 给对象添加 getter,setter，用于依赖收集(getter)和派发更新(setter),会给响应式数据添加__ob__属性。
