# 设计模式

# SOLID 原则

S 单一指责原则 O 开放封闭原则: 扩展开放，修改封闭,需求变化时候通过扩展来解决 L 李氏置换原则 I 接口隔离原则 D 依赖倒置原则: 函数返回尽量为接口类型

# 工厂模式

当遇到一个 class 的时候，可以考虑采用工厂模式，定义一个工厂函数即可,返回一个 new 之后的对象

场景：

  1. $
  2. _createElementVNode

优点： 1.通过创建一个个工厂函数，可以方便的传入参数去做同类的功能。 2.将对象的创建封装，实现封装和隔离。

# 单例模式

# 原型模式

主要考虑前端相关的原型。 Object.create 将生成对象的proto指向参数

# 代理模式

更加方便，更加安全,避免敏感信息

场景：

  1. dom 事件代理
  2. webpack devServer proxy
  3. nginx 代理
  4. vue3 数据响应

Proxy 使用场景：

  1. 跟踪属性访问
  2. 隐藏属性
  3. 验证属性
  4. 记录实例

# 装饰器模式

动态添加功能，不影响原有的功能

# 观察者模式

一般包含一个主题， 多个观察者 场景：

  1. dom 事件，定时,Promise.then
  2. vue 生命周期(data 相当于一个主题，watcher 相当于观察者。当 data 变化的时候，会 notify watcher，然后 watcher 就会 re-render 组件)

# 发布订阅模式

相对于观察者模式，多了一个主动触发操作

场景：1.自定义事件 2.postMessage 通讯

自定义事件需要及时销毁

# 迭代器模式

ps: for 循环不是迭代器

顺序访问有序结构，不知道数据的长度和内部结构，高内聚，低耦合

场景：

  1. 有序结构
  2. Symbol.iterator 和迭代器
  3. 遍历 dom 树

** 迭代器的具有方法 next(),和 generator 函数一致

作用：

  1. 只要有迭代器，就可以用 for...of 来遍历获取子元素
  2. 解构，Array...from
  3. 创建 Map 和 Set

# 职责链模式

多个角色处理一个流程

场景：

  1. jquery 链式操作
  2. promise
