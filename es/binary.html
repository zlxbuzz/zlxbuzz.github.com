    # js 的二进制

    # Blob

    二进制对象

    File 对象：继承自 Blob 对象,比如 input[type="file"]。

    方法：

    1. 文件下载: URL.createObjectURL(blob),给 a 标签,(ie 可以用 window.navigator.msSaveBlob)
    2. 图片显示: URL.createObjectURL(File_Blob)，给 img[src]
    3. blob.slice 支持资源分段上传
    4. 本地读取文件：FileReader 可以将 Blob 或者 File 转化问 ArrayBuffer/文本/Data 等数据

    ps: window.URL.revokeObjectURL(blob),尽量再不需要的时候销毁

    Blob 生产的 URL 很短，可以理解为是对浏览器自身的一个协议,内部引用

    new Blob(/ArrayBuffer(二进制数据缓冲区), ArrayBufferView,Blob,DOMString/)

    # ArrayBuffer

    # 二进制缓冲区,相比 Blob 可以更具体修改二进制数据(需要 TypeArray/DataView),更底层
    # 生成 arrayBuffer: ajax 设置 xhr.responseType = "arraybuffer" ， fileReader.readAsArrayBuffer(file)
    1. 转 Blob: new Blob([buffer])
    2. 转字符串或者 json：
    const blob = new Blob([buffer]);
const fileReader = new FileReader();
fileReader.onload = function() {
    const result = fileReader.result;
    console.log(result); // json需要parse一下
};
fileReader.readAsText(blob);

    ArrayBuffer 类似数组，但也有不同：

    1. 初始化后大小固定
    2. 数据在栈中
    3. 没有 push 等方法
    4. 只能读，写需要 TypeArray/DataView

    TypeArray(类型数组):

    # Uint8Array 数组类型表示一个8位无符号整型数组，创建时内容被初始化为0
# Int8Array 类型数组表示二进制补码8位有符号整数的数组

const typedArray1 = new Int8Array(8);
typedArray1[0] = 32;

const typedArray2 = new Int8Array(typedArray1);
typedArray2[1] = 42;

console.log(typedArray1);
//  output: Int8Array [32, 0, 0, 0, 0, 0, 0, 0]

console.log(typedArray2);
//  output: Int8Array [32, 42, 0, 0, 0, 0, 0, 0]

    DataView:

    const buffer = new ArrayBuffer(16);
const view = new DataView(buffer); // DataView 视图是一个可以从 二进制ArrayBuffer 对象中读写多种数值类型的底层接口
view.setInt8(2, 42);
console.log(view.getInt8(2));
// 输出: 42

    # Buffer (Node.js)

    Node.js 的二进制缓冲区

    let buffer = Buffer.concat(chunks);
console.log(buffer.toString());

    # FileReader

    可以读取 Blob 或者 File 转换为其他：

    FileReader.readAsText(Blob)：将Blob转化为文本字符串
FileReader.readAsArrayBuffer(Blob)： 将Blob转为ArrayBuffer格式数据
FileReader.readAsDataURL(Blob): 将Blob转化为Base64格式的Data URL
