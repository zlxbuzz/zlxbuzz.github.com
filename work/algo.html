          # 算法学习

          # 二分算法

          输入必须有序: 对于包含 n 个元素的列表，采用二分法最多需要 log2 n 步，简单查找需要 n 步.

          用大 O 表示法来表示：括号内为操作数，指出了算法运行时间的增速,代表最糟糕情况。大 O 表示法会省略常量数字 二分查找: O(log n) 简单查找：O(n) 快排: O(n * log n) 选择排序: O(n x n) 旅行商问题: O(n!)

          # 选择排序

          # 数组: 读取操作有优势

          # 链表：插入元素有优势

          # 递归

          在性能上递归可能没有循环好，但是更加容易理解

          递归包含：基线条件（函数不调用自己）和递归条件（函数调用自己）

          # 快速排序

          实现快速排序时，随机地选择用作基准值的元素。快速排序的平均运行时间为 O(n log n)

          # 图

          图模拟一组连接，由节点和边组成，一个节点可能由多个边。

          # 广度优先搜索

          用于图的，解决最短路径问题的算法被称为广度优先搜索,先查找一度关系，在查找二度关系，以此类推。运行时间为 O(节点数+边数) O(V+E)

          # 树

          树是一种特殊的图，没有往后指的边

          # 狄克斯特拉算法

          找出最快的路径,也就是总权重最小的路径（而非段数最少的路径）,只适用于有向无环图 (1) 找出最便宜的节点，即可在最短时间内前往的节点。

          (2) 对于该节点的邻居，检查是否有前往它们的更短路径，如果有，就更新其开销。

          (3) 重复这个过程，直到对图中的每个节点都这样做了。

          (4) 计算最终路径。

          带权重的图也叫做加权图 如果有负权重，则需要用贝尔曼福德算法

          # 贪婪算法

          并集：集合合并 交集：共有的元素 差集：从一个集合剔除另一个集合的元素

          # 动态规划

          先解决子问题，在逐步解决大问题
